# GUD Scout - 2026-04-19

## Hallazgos principales

- **hugohe3/ppt-master** (73★, Python) — INSTALAR — IA genera PPTX nativamente editables desde cualquier documento; ideal para entregar decks de consultoría sin intervención manual.
- **lewislulu/html-ppt-skill** (48★, HTML) — INSTALAR — AgentSkill con 24 temas y 31 layouts para generar presentaciones HTML directamente desde el agente, sin salir del flujo.
- **addyosmani/agent-skills** (100★, Shell) — INSTALAR — Colección de skills de producción para agentes IA; referencia directa para crear skills propias del gud-framework.
- **thedotmack/claude-mem** (201★, TypeScript) — VIGILAR — Plugin que captura contexto automáticamente en Claude Code; podría ser la capa de memoria persistente por proyecto de cliente.
- **opendataloader-project/opendataloader-pdf** (45★, Java) — VIGILAR — Parser PDF para datos listos para IA; automatiza la ingesta de documentos de clientes en el pipeline.
- **multica-ai/multica** (138★, TypeScript) — VIGILAR — Convierte agentes en compañeros de equipo con memoria y roles; patrón útil para equipos multi-agente en flujos de consultoría.
- **midudev/autoskills** (38★, Shell) — VIGILAR — Un solo comando instala todo el stack de skills IA; gestión de dependencias del ecosistema gud-framework.

## Para tener en el radar

- **rtk-ai/rtk** (65★, Rust) — Proxy CLI que reduce consumo de tokens LLM en 60%; relevante para bajar costos operativos del framework.
- **MemPalace/mempalace** (53★, Python) — Sistema de memoria IA open source benchmarkeado; alternativa a claude-mem para persistir contexto entre sesiones.
- **lsdefine/GenericAgent** (159★, Python) — Bucle de agente para automatización de escritorio; ideas para automatizar flujos en herramientas de oficina de clientes.
- **sickn33/antigravity-awesome-skills** (34★) — Colección 800+ skills agénticas; directorio de referencia para encontrar skills ya construidas antes de crear desde cero.

## Tendencias detectadas

- **Explosión de skills como unidad de distribución**: la mayoría de repos trending son colecciones de skills o frameworks para gestionarlas; el ecosistema se está fragmentando en skills especializadas en lugar de agentes monolíticos.
- **Presentaciones como output de agente**: dos repos independientes de PPTX/HTML-PPT aparecen trending el mismo día, señal de demanda real de generación automatizada de decks.
- **Memoria persistente como problema resuelto**: claude-mem y mempalace apuntan a que la retención de contexto entre sesiones ya tiene soluciones maduras listas para integrar.
