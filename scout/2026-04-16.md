## Hallazgos principales

- **googleworkspace/cli** (983 ⭐ semana, Rust) — INSTALAR — CLI oficial de Google Workspace: automatiza Drive, Docs y Slides desde terminal, ideal para pipelines de entrega de consultoria.

- **opendataloader-project/opendataloader-pdf** (150 ⭐, Java) — INSTALAR — Parser PDF orientado a IA; permite ingestar propuestas, contratos y reportes de cliente directamente en agentes.

- **paperclipai/paperclip** (3.053 ⭐ semana, TypeScript) — VIGILAR — Orquestacion para tareas "zero-human"; podria manejar flujos de aprobacion y entregables sin intervencion manual.

- **addyosmani/agent-skills** (155 ⭐, Shell) — INSTALAR — Skills de produccion para agentes Claude Code; modular y listo para enchufar en gud-framework como capa de capacidades.

- **rtk-ai/rtk** (634 ⭐ semana, Rust) — INSTALAR — Proxy CLI que reduce el consumo de tokens LLM un 60%; reduce costos operativos directamente para agentes de consultoria de alto volumen.

- **rowboatlabs/rowboat** (162 ⭐, TypeScript) — VIGILAR — AI coworker open-source con memoria persistente; candidato para portal de cliente con historial de proyecto integrado.

- **multica-ai/multica** (240 ⭐, TypeScript) — VIGILAR — Convierte agentes en compañeros de equipo con coordinacion multi-agente; util para orquestar especialistas (finanzas, marketing, ops) en una consultoria.

- **vectorize-io/hindsight** (75 ⭐, Python) — VIGILAR — Memoria de agente que aprende con el tiempo; permite que gud-framework recuerde contexto de cada cliente entre sesiones.

- **promptfoo/promptfoo** (1.006 ⭐ semana, TypeScript) — INSTALAR — Testing de prompts, agentes y RAGs; control de calidad esencial antes de entregar automatizaciones al cliente.

- **coreyhaines31/marketingskills** (60 ⭐, JavaScript) — VIGILAR — Skills de marketing para Claude Code; acelera generacion de propuestas comerciales y materiales de cliente.

## Tendencias detectadas

- **Explosion de skills reutilizables**: El ecosistema se esta fragmentando en skills especializadas (marketing, obsidian, produccion). gud-framework deberia tener su propio conjunto de skills publicadas.
- **CLI como interfaz de automatizacion**: Google Workspace CLI + rtk + paperclip indican que el terminal es el nuevo centro de integracion. Las pipelines sin UI son la norma.
- **Memoria de agente como diferenciador**: hindsight, rowboat y multica convergen en persistencia de contexto. El cliente que recuerda tiene ventaja competitiva.
- **Reduccion de costos de tokens**: rtk y JuliusBrussee/caveman apuntan a eficiencia de tokens como preocupacion real. Relevante para escalar gud a multiples clientes.
