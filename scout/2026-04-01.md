# GUD Scout - 2026-04-01

## Hallazgos principales

- **paperclipai/paperclip** (+193, TypeScript) — VIGILAR — Orquestación open-source para "empresas sin humanos"; el pipeline de automatización completa es exactamente lo que gud-framework necesita para workflows de consultoría sin intervención manual.

- **phuryn/pm-skills** (+43) — INSTALAR — 100+ skills agénticos de gestión de proyectos listos para Claude Code; pueden conectarse directamente al harness de gud-framework para tareas como OKRs, seguimiento de entregables y reportes de cliente.

- **Yeachan-Heo/oh-my-claudecode** (+89, TypeScript) — VIGILAR — Orquestación multi-agente "teams-first" para Claude Code; permite coordinar subagentes especializados (analista, redactor, presentador) dentro de un mismo engagement de consultoría.

- **NousResearch/hermes-agent** (+175, Python) — VIGILAR — Agente que "crece contigo" con memoria y adaptación continua; base sólida para construir un agente consultor que aprenda del historial de cada cliente.

- **thedotmack/claude-mem** (+69, TypeScript) — INSTALAR — Plugin que captura automáticamente el contexto de sesión en Claude Code; resuelve el problema de pérdida de contexto entre sesiones largas de proyecto.

- **revfactory/harness** (+39, HTML) — INSPIRACION — Meta-skill que diseña templates de agentes por dominio; el patrón es útil para crear harnesses especializados (consultoría estratégica, M&A, due diligence).

- **gsd-build/get-shit-done** (+71, JavaScript) — VIGILAR — Meta-prompting ligero con gestión de contexto; podría servir como capa de orquestación de prompts para las fases del ciclo de consultoría (discovery, análisis, entrega).

- **Fission-AI/OpenSpec** (+37, TypeScript) — VIGILAR — Desarrollo guiado por especificaciones para asistentes IA; aplicable a definir contratos de entregables de consultoría que el agente respeta automáticamente.

- **EveryInc/compound-engineering-plugin** (+38, TypeScript) — VIGILAR — Plugin Office para Claude Code; abre la puerta a generar PPTX y documentos Word directamente desde workflows de agente.

- **ComposioHQ/awesome-claude-skills** (+39, Python) — INSTALAR — Colección curada de skills Claude con integraciones externas (CRMs, APIs); catálogo de conectores listos para integrar en gud-framework.

- **kepano/obsidian-skills** (+35) — Gestión de conocimiento con agentes; patrón interesante para base de conocimiento de consultoría.
- **mvanhorn/last30days-skill** (+72, Python) — Skill que investiga cualquier tema en Reddit/web; útil para research de sector/cliente previo a un engagement.
- **ruvnet/ruflo** (+42, TypeScript) — Plataforma de orquestación líder para Claude; alternativa a evaluar vs arquitectura actual de gud-framework.
- **Crosstalk-Solutions/project-nomad** (+37, TypeScript) — Suite de proyecto offline self-contained; referencia para modo sin conexión en deployments on-premise de consultoría.

## Tendencias detectadas

- **Explosión de skills/harnesses**: El ecosistema se fragmenta en cientos de skill-packs (phuryn, sickn33, ComposioHQ, MiniMax, VoltAgent). gud-framework debería definir pronto qué skills instala por defecto.
- **Multi-agente como estándar**: oh-my-claudecode, paperclip, ruflo y hermes-agent muestran que la orquestación de equipos de agentes (no solo uno) es el patrón dominante.
- **Memoria persistente como necesidad crítica**: claude-mem y hermes-agent priorizan la continuidad de contexto; sin esto, los workflows largos de consultoría pierden coherencia.
- **Office/documentos en el radar**: compound-engineering-plugin y GLM-OCR (+59) apuntan a que la generación de entregables en formatos Office está siendo abordada activamente.
